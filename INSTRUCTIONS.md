# Project Instructions

Welcome to the AgentX Hackathon repository!

Please follow the instructions & submission guidelines below to set up and document your project appropriately:

## 1. README.md

- Include all project details in the `README.md` file.
  - Project name
  - Product name
  - Team name
  - About the project
  - Steps to run the project
  - Relevant constraints for running the project
- Use the provided dummy examples as placeholders and update them with your actual project details.

## 2. Demo Recording

- Record a short demo of your project.
- Place the demo recording under the `demo` folder at the root of the repository.
- Use the provided dummy example as a placeholder if needed and replace it with your own demo.

## 3. ARCHITECTURE.md

- Add all architecture-related documentation to the `ARCHITECTURE.md` file.
  - Include physical, logical, and deployment architecture diagrams and descriptions.
  - List all cloud components used or planned.
- Use the dummy examples provided as placeholders and update them as you finalize your architecture.

## 4. General Guidance

- All placeholders or dummy examples in the template files are for reference. Make sure to replace them with your actual project details.
- Ensure all files are up-to-date and accurately represent your project before final submission.

## 5. Getting Started

- Participants will be onboarded on Deloitte-US-DEP-Training Github org
- Login with your `<username>_deloitte` github account. So e.g. `ansriraman_deloitte`
- Platform specific Github repo templates
  - AWS, GCP, Azure
- Choose your appropriate template as a starting point
- Create a fork of your chosen platform's template repo. Repo name - `agentx-hackathon-<team name>`
- Template contains AEF accelerators pre-configured
  - LLM Gateway, Devals, N8n
- Build your agentic solutions using any framework of choice: Langgraph, Crew.ai, ADK, etc
- Use Github codespaces to get access to preconfigured LLM gateway + access to LLMs
- Preprovisioned access to various cloud LLMs and Embedding models
  - AWS Bedrock LLM, GCP Gemini models, Azure OpenAI Models
- No access to Cloud account
- You are not expected to deploy your solution to any cloud. All demos to be done locally.
